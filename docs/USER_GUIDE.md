# ChipWise Enterprise — 用户指南

**目标读者**：使用 ChipWise Enterprise 进行芯片数据查询、BOM 审查和测试用例生成的硬件工程师。
**更新日期**：2026-04-13

---

## 快速开始

打开浏览器，访问 `http://localhost:7860`（或管理员提供的 URL）。

---

## 1. 登录与账户管理

### 本地登录

1. 打开 ChipWise 网页应用
2. 点击**登录**，输入用户名和密码
3. 点击**确认登录** — 您将获得一个有效期为 30 分钟的访问令牌（活跃操作时自动刷新）

### SSO 登录（Keycloak / DingTalk / Feishu）

如果您的组织使用单点登录（SSO）：
1. 在登录页面点击**通过 [您的身份提供商] 登录**
2. 页面将跳转至您组织的身份认证服务
3. 使用企业凭证登录
4. ChipWise 会自动创建或更新您的账户（即时供应）

### 角色与权限

| 角色 | 访问权限 |
|---|---|
| `viewer` | 只读：查询、对比、搜索 |
| `user` | 所有 viewer 权限 + BOM 上传、知识库编辑 |
| `admin` | 所有 user 权限 + 系统监控、用户管理 |

### 修改密码

点击右上角个人头像 → **修改密码**。最低要求：8 个字符，至少包含 1 个大写字母和 1 个数字。

## 2. 聊天 (Chat) — 自然语言查询

**聊天 (Chat)** 标签页是查询芯片数据的主要界面。

### 基本查询

使用自然语言输入问题，然后按 **Enter** 或点击**发送**：

```
STM32F407VG 的最高工作温度是多少？
```

```
对比 STM32F4 和 STM32H7 系列的 ADC 分辨率
```

```
哪些 NXP 微控制器支持 CAN-FD 且 Flash 大于 512KB？
```

### 获得更好结果的技巧

| 技巧 | 示例 |
|---|---|
| 明确指定芯片型号 | 使用 "STM32F407VG" 而非仅 "STM32" |
| 在问题中包含单位 | "供电电压范围（单位：伏特）" |
| 明确要求对比 | "对比 X 和 Y 的 [参数]" |
| 提及应用场景 | "用于车规级" 会自动添加相关勘误过滤 |

### 理解响应内容

每个响应都包含**引用** — 可点击的源文档链接（数据手册、参考手册），用于支撑回答内容。响应中的 `[Doc1]`、`[Doc2]` 标记对应答案下方的引用列表。

### 流式响应

较长的回答会逐 Token 实时流式输出，您可以在生成过程中阅读响应内容。如需停止生成，请点击**停止**。

### 对话历史

每个会话保留最多 10 轮对话历史。Agent 会利用之前的对话作为上下文 — 您可以提出后续问题：
```
STM32F407VG 的主频是多少？
→ [回答：168 MHz]

它的 Flash 容量呢？
→ [正确跟进，回答 STM32F407VG 的信息]
```

点击**新建对话**可开始一个不含先前上下文的全新会话。

## 3. 芯片对比 (Compare)

**对比 (Compare)** 标签页提供结构化的并排对比功能。

### 如何对比

1. 打开**对比 (Compare)** 标签页
2. 输入芯片型号，以逗号分隔：
   ```
   STM32F407VG, STM32H743VI
   ```
3. 可选：指定要关注的参数：
   ```
   ADC 分辨率、Flash 容量、工作温度、价格
   ```
4. 点击**对比**

### 查看结果

结果以 Markdown 表格形式呈现：

| 参数 | STM32F407VG | STM32H743VI |
|---|---|---|
| 内核 | Cortex-M4 @ 168 MHz | Cortex-M7 @ 480 MHz |
| Flash | 1 MB | 2 MB |
| ADC | 12-bit, 2.4 MSPS | 16-bit, 3.6 MSPS |
| 工作温度 | -40°C to +105°C | -40°C to +125°C |

### 国产替代推荐

对比完成后，ChipWise 会自动推荐国产替代方案（来自内置的芯片替代数据库），显示在主对比表下方。

## 4. BOM 审查 (BOM Review)

**BOM 审查 (BOM Review)** 标签页用于分析物料清单中的风险和问题。

### 支持的文件格式

上传 Excel 文件（`.xlsx`），包含以下列（表头灵活匹配 — ChipWise 自动识别常见列名）：

| 列名 | 是否必填 | 说明 |
|---|---|---|
| 料号 (Part Number) | 是 | 芯片标识符（如 STM32F407VG） |
| 数量 (Quantity) | 否 | 用量 |
| 制造商 (Manufacturer) | 否 | 芯片制造商名称 |
| 供电电压 (Supply Voltage) | 否 | 标称供电电压（如 3.3V） |

### 执行 BOM 审查

1. 点击**上传 BOM** 并选择 `.xlsx` 文件
2. 点击**开始审查**
3. 等待分析完成（通常需要 30–120 秒，取决于 BOM 规模）

### 理解审查报告

审查报告包含三个部分：

**停产 / 不推荐新设计标记 (EOL / NRND)**
已停产或不推荐用于新设计的元器件会以红色高亮显示，应在量产前进行替换。

**设计冲突**
BOM 与芯片数据手册规格之间的供电电压或电气参数冲突。请仔细审查 — 这可能意味着 PCB 设计存在错误。

**替代建议**
对于每个被标记的元器件，ChipWise 会推荐有现货的国产替代方案，规格相当或更优。

### 下载报告

点击**下载报告 (Excel)** 将完整审查结果保存为 `.xlsx` 文件，方便发送给采购团队。

## 5. 芯片选型

在**聊天 (Chat)** 标签页中使用选型查询，或直接提问：

```
帮我找一款 ARM Cortex-M4 MCU，Flash 至少 512KB，支持 USB OTG，
工作温度 -40°C 到 125°C，厂商为 STMicroelectronics
```

ChipWise 将会：
1. 将您的需求解析为筛选条件
2. 使用参数化过滤器查询芯片数据库
3. 按相关性重新排序结果
4. 展示前 5 个候选芯片及其关键规格
5. 如适用，推荐国产替代方案

### 高级筛选

如需精确筛选，可使用结构化语言：
```
MCU: core=Cortex-M4, flash >= 512KB, RAM >= 128KB, 
     package=LQFP64, temp_range includes -40 to 125, 
     has: USB, CAN, I2S
```

## 6. 测试用例生成

在聊天 (Chat) 标签页中要求 ChipWise 为指定芯片生成测试用例：

```
为 STM32F407VG 的 ADC 外设生成测试用例
```

```
为 NXP LPC1769 的 USB Full Speed 创建测试用例列表
```

ChipWise 基于芯片的数据手册、参考手册和已知勘误生成结构化测试用例。输出内容包括：
- 测试用例 ID 和名称
- 前置条件和环境搭建
- 测试步骤
- 预期结果
- 通过/失败判定标准

### 导出测试用例

生成完成后，在聊天响应中点击**导出为 Excel**，将测试用例下载为 `.xlsx` 文件，包含：
- 工作表 1：测试用例表（ID、名称、前置条件、步骤、预期结果）
- 工作表 2：元数据（芯片型号、生成日期、模型版本）

## 7. 知识库 (Knowledge Base)

**知识库 (Knowledge Base)** 标签页是团队共同维护的工程笔记、经验总结和设计技巧仓库。

### 添加笔记

1. 打开**知识库 (Knowledge Base)** 标签页
2. 点击**新建笔记**
3. 填写以下信息：
   - **标题**：简短描述（如 "STM32F4 高温下的 ADC 精度问题"）
   - **内容**：详细笔记内容（支持 Markdown）
   - **芯片 ID**（可选）：将笔记关联到特定芯片
   - **标签**（可选）：逗号分隔的标签，用于分类（如 `ADC, thermal, production-issue`）
4. 点击**保存**

### 搜索笔记

在**搜索**框中输入关键词。ChipWise 使用语义搜索 — 无需精确匹配关键词：
```
高温下的 ADC 噪声
→ 返回关于"热漂移"、"接近 Tmax 时的 ADC 精度"等相关笔记
```

可通过下拉筛选器按芯片 ID 或标签过滤。

### 编辑和删除笔记

点击铅笔图标编辑笔记，或点击垃圾桶图标删除。只有笔记作者和管理员可以编辑或删除笔记。

## 8. 系统监控 (Monitor)（仅管理员）

管理员可通过**监控 (Monitor)** 标签页查看系统状态。

**服务健康状态**：实时显示全部 7 个系统服务的状态（FastAPI、LM Studio、BGE-M3、bce-reranker、PostgreSQL、Milvus、Redis）。

**Token 用量**：按模型和角色分类的每日 LLM Token 消耗明细，可用于容量规划。

**API 版本**：当前部署版本和运行时间。

如果某个服务显示 `degraded`（降级）或 `unavailable`（不可用），请联系系统管理员。

## 9. 常见问题 (FAQ)

**问：我的查询非常慢（超过 30 秒），是什么原因？**
主推理模型（35B）在处理复杂的多步推理查询时可能需要 15–30 秒，这是正常现象。如果持续超时，请联系管理员检查 LM Studio 状态。

**问：响应显示"未找到关于 [某芯片] 的信息"，我该怎么办？**
该芯片可能尚未录入数据库。请联系管理员通过**文档上传**（管理员功能）导入数据手册，或通过爬虫从制造商网站导入。

**问：我能查询不在数据库中的芯片吗？**
不能 — ChipWise 仅基于已导入的文档进行回答，不会为未收录的芯片编造规格参数。这是刻意设计的，旨在防止提供不正确的技术信息。

**问：数据有多新？**
数据时效性取决于文档最后一次导入的时间。爬虫每晚（凌晨 2 点）会自动抓取已配置的制造商网站。您可以在引用来源中查看文档的导入日期。

**问：我的 BOM 有 500 个元器件，ChipWise 会全部审查吗？**
会的，但可能需要几分钟。大型 BOM 会分批处理，您会看到进度指示器。您可以离开页面稍后再回来查看 — 结果会保存在您的会话中。

**问：谁能看到我的查询记录和知识库笔记？**
查询记录绑定在您的用户账户下，仅您本人和系统管理员可见。知识库笔记对您组织内的所有用户可见。请勿在共享知识库笔记中添加机密项目信息。

**问：对比表中缺少某些参数，我能添加吗？**
可以 — 在**对比 (Compare)** 标签页中，将具体参数名称添加到"对比参数"字段。如果该参数存在于已导入的数据手册中，就会被包含在对比结果中。

**问：我在监控 (Monitor) 标签页看到某个服务显示 "degraded"（降级），需要担心吗？**
降级表示该服务响应缓慢或存在问题，但系统仍在以降低的能力运行（例如重排序功能被禁用、缓存无响应）。请联系管理员处理 — 请勿自行重启服务。
